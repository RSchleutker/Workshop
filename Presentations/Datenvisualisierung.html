ggplot2 - Layered Grammar of Graphics

Die Idee

Die Idee ist, jedes Diagramm nach dem gleichen Schema aufzubauen und nach und nach die Komponenten hinzuzufgen.

  • ggplot2: Elegant Graphics for Data Analysis: Gibt ein tieferes Verstndnis von ggplot2.

Konkret geht es um folgende Aspekte:

  • **Data:** Welche Daten sollen dargestellt werden? Nicht so einfach, wie man denkt.
  • Aesthetic mapping: Auf welcher Achse soll welche Variable dargestellt werden? Es gibt mehr als nur x- und y-Achse (Farbe, Gre, Transparenz, Form, …).
  • Scales:: WIE soll die jeweilige Achse skaliert sein? Logarithmisch, Farbpalette, …
  • Statistical transformation: Sollen die Rohdaten dargestellt werden oder prozessierte Daten: Regressionslinien, Boxplots, Histogramme, …
  • Geometric Objects: In welche Form sollen die Daten dargestellt werden? Z.B. Linien, Punkte, Balken, …
  • Positional adjustments: Sollen Sulen z.B. ber- oder nebeneinander dargestellt werden? Kartesisches oder Polarkoordinatensystem? Facetting?

Aesthetic Mapping

Dies ist einer der wichtigsten Punkte!

  • Datenstze bestehen meist aus mehreren Variablen. Welche Variable wird auf welcher Skala dargestellt!
  • Etwas klarer: Einen groen Wert kann man nicht nur dadurch darstellen, dass er sich sehr weit rechts (x-Achse) oder oben (y-Achse) auf einer rumlichen Skala befindet. Ein groer Wert kann auch
    • rot sein, whrend kleine Werte blau sind.
    • deckend sein, whrend kleine Werte transparent sind.
    • ein groer Punkt sein, whrend kleine Werte kleinere Punkte sind.
  • Das gleiche funktioniert natrlich auch mit diskreten Variablen.

Nun kommt der Plot, Schritt fr Schritt

Jedes Diagramm startet mit der Funktion ggplot().

  • Das erste Argument ist IMMER data.
ggplot(data = iris)

Der Plot: Die Mappings

Die Variablen werden ber das Argument mapping = aes() auf die Achsen gemapped.

  • Das zweite Argument ist IMMER mapping.
ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species))

Der Plot: Die Geometric Objects

Weitere Eigenschaften knnen ber ein + hinter der letzten Zeile hinzugefgt werden!

Geometric objects fangen IMMER mit geom_ an.

ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species)) +
  geom_point()

Der Plot: Statistical Transformations

Statistical Transformations kann man immer ber stat_ hinzufgen.

ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species)) +
  geom_point() +
  stat_smooth(method = "lm")

Der Plot: Scales

ber Scales kann man alles ndern, was mit Achsen etc. zu tun hat. Sie werden immer ber scale_ aufgerufen!

ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species)) +
  geom_point() +
  stat_smooth(method = "lm") +
  scale_x_continuous(name = "Sepallnge (cm)", limits = c(0,NA)) +
  scale_y_continuous(name = "Sepalbreite (cm)", limits = c(0,NA)) +
  scale_colour_discrete(name = "Spezies", labels = c("Spezies 1", "Spezies 2", "Spezies 3"))

Positional Adjustments

Hier schauen wir uns fr den moment nur das Facetting an:

ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species)) +
  geom_point() +
  stat_smooth(method = "lm") +
  scale_x_continuous(name = "Sepallnge (cm)", limits = c(0,NA)) +
  scale_y_continuous(name = "Sepalbreite (cm)", limits = c(0,NA)) +
  scale_colour_discrete(name = "Spezies", labels = c("Spezies 1", "Spezies 2", "Spezies 3")) +
  facet_wrap(~ Species, ncol = 3)

Das Theme

Zuletzt kann man dem Plot noch ein anderes Standard-Theme geben und gleichzeitig die Textgre anpassen:

ggplot(data = iris, mapping = aes(x = Sepal.Length, y = Sepal.Width, colour = Species)) +
  geom_point() +
  stat_smooth(method = "lm") +
  scale_x_continuous(name = "Sepallnge (cm)", limits = c(0,NA)) +
  scale_y_continuous(name = "Sepalbreite (cm)", limits = c(0,NA)) +
  scale_colour_discrete(name = "Spezies", labels = c("Spezies 1", "Spezies 2", "Spezies 3")) +
  facet_wrap(~ Species, ncol = 3) +
  theme_bw(base_size = 9)

Den Plot speichern

Den Plot kann man anschlieend mit der ggsave()-Funktion speichern:

ggsave(filename = "./Results/Plot.png",
       plot = last_plot(),
       width = 80, height = 60,
       units = "mm",
       dpi = 600)

Was es beim speichern zu beachten gibt

  • Wenn mglich, den Plot als Vektorgrafik speichern: filename = ".../Plot.pdf" oder filename = ".../Plot.svg".
  • Soll die Auflsung bei Rastergrafiken erhht werden, wird dies ber das Argument dpi gemacht. NIEMALS ber width und/oder height.
  • Messt in euren Dokumenten mal nach, wie breit der Textbereich ist und orientiert die Gre des Plots daran. So ist die Schrift in allen Diagrammen im spteren Protokoll gleich gro.
